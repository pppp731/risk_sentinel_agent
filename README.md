# 风险哨兵 // RISK SENTINEL

基于NLP技术的企业风险监测与预警系统 - 智能舆情监控平台

## 📋 项目简介

**风险哨兵**是一个智能化的企业风险监测系统，通过自然语言处理(NLP)技术自动采集多源新闻数据，实时分析企业舆情风险，提供可视化风险报告和历史追踪功能。

系统支持多源新闻聚合（NewsAPI、GNews、RSS订阅），自动进行情感分析和风险识别，生成综合风险评分，并提供历史记录存档和报告导出功能。

## ✨ 主要功能

### 1. 多源新闻采集
- **NewsAPI** - 国际新闻数据源（需API Key）
- **GNews API** - 国内可访问的全球新闻（需API Key）
- **RSS订阅** - 自动抓取CNBC、MarketWatch、36氪等权威媒体
- **智能去重** - 基于标题相似度自动去重（阈值80%）

### 2. 智能风险分析
- **情感分析** - 使用TextBlob判断正面/负面/中性情感
- **风险识别** - 基于关键词匹配识别6大类风险：
  - 🔴 **劳工风险** - 罢工、劳资纠纷、员工抗议等
  - 🔴 **法律风险** - 诉讼、罚款、监管调查等
  - 🔴 **供应链风险** - 召回、质量问题、生产中断等
  - 🔴 **财务风险** - 亏损、债务、股价下跌等
  - 🔴 **声誉风险** - 丑闻、抵制、公关危机等
  - 🔴 **环境风险** - 污染、环保违规等
- **风险评分** - 综合计算0-100分风险评分，自动分级

### 3. 历史记录追踪 ⭐NEW
- 自动保存每次分析结果到本地JSON存储
- 侧边栏展示最近5条历史记录
- 点击历史记录可查看详情对比
- 支持最多保存50条历史记录

### 4. 可视化报告
- **仪表盘** - 展示关键指标（文章数、风险分、风险等级、负面占比）
- **风险预警** - 高/中/低三级预警提示
- **情感分布** - 可视化展示正面/负面/中性文章比例
- **风险类型分布** - 条形图展示各类风险出现频次
- **文章列表** - 展示检测到的文章标题、情感标签和风险分数

### 5. 导出报告 ⭐NEW
- 支持将分析结果导出为JSON格式
- 支持历史记录对比分析

## 🚀 快速开始

### 环境要求

- Python 3.7+
- 网络连接（用于获取新闻数据）

### 安装步骤

1. **克隆/下载项目**
```bash
cd risk_sentinel
```

2. **安装依赖**
```bash
pip install -r requirements.txt
```

> **国内用户加速**：如果安装慢，可以使用清华镜像：
> ```bash
> pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
> ```

```bash
python app.py
```

在浏览器中打开 http://localhost:5000 即可使用。

## 🔧 配置说明

### 环境变量设置

编辑 `config.py` 文件配置API密钥：

```python
# ==================== API 密钥配置 ====================

# NewsAPI 密钥（可选，用于获取国际新闻）
# 1. 访问 https://newsapi.org/ 注册免费账号
# 2. 获取 API Key（免费版 100 次/天）
NEWSAPI_KEY = "your_newsapi_key_here"

# GNews API 密钥（推荐，国内可访问）
# 1. 访问 https://gnews.io/ 注册免费账号
# 2. 获取 API Key（免费版 100 次/天）
GNEWS_API_KEY = "your_gnews_api_key_here"
```

### 自定义监控关键词

修改 `config.py` 中的搜索关键词：

```python
# 目标公司名称（GNews 简单查询效果更好）
COMPANY_KEYWORDS = "coconut water"

# 或者使用复杂查询（NewsAPI支持）
# COMPANY_KEYWORDS = "Apple OR iPhone OR 苹果"
```

### 添加RSS订阅源

在 `config.py` 的 `RSS_SOURCES` 列表中添加新的RSS源：

```python
RSS_SOURCES = [
    {
        "name": "媒体名称",
        "url": "https://example.com/rss.xml"
    },
    # 添加更多...
]
```

## 🎯 API接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/` | GET | 主页面 |
| `/api/analyze` | GET | 执行风险分析 |
| `/api/history` | GET | 获取历史记录列表（支持`?limit=5`参数） |
| `/api/history/<id>` | GET | 获取单条历史详情 |
| `/api/history/save` | POST | 保存当前分析到历史 |
| `/api/health` | GET | 健康检查 |

## 🛠️ 技术特点

1. **多源数据聚合** - 支持API和RSS混合采集，提高数据覆盖率
2. **智能去重** - 使用SequenceMatcher计算标题相似度，自动去重
3. **模块化设计** - 新闻获取、风险分析、历史管理分离，易于扩展
4. **响应式界面** - 支持桌面和移动端访问
5. **本地存储** - 历史记录本地JSON存储，无需数据库

## ❓ 常见问题

**Q: 为什么GNews获取不到数据？**

A: 检查以下几点：
1. 确认API Key已正确配置在 `config.py` 中
2. GNews免费版仅支持简单查询词（如"coconut water"），复杂OR查询可能返回0条
3. 检查网络连接，GNews在国内可直接访问

**Q: 如何添加更多新闻源？**

A: 在 `config.py` 的 `RSS_SOURCES` 列表中添加RSS订阅地址，格式：
```python
{"name": "媒体名称", "url": "https://example.com/rss.xml"}
```

**Q: 历史记录保存在哪里？**

A: 历史记录保存在 `history_data/` 目录下：
- `records.json` - 记录索引（最多50条）
- `detail_*.json` - 详细分析数据

**Q: 如何清空历史记录？**

A: 删除 `history_data/` 目录下的所有文件即可。
